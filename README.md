# Автоматизированная система обработки корпоративной почты

Система читает входящие письма из папки `inbox/`, классифицирует их по категориям и раскладывает по папкам. Формирует лог и статистику по результатам обработки.

## Структура проекта

```
mail_processor/
├── inbox/              # Папка с входящими письмами (кладёте сюда)
├── processed/          # Сюда попадут письма после обработки (создаётся автоматически)
│   ├── critical/       # Критические инциденты
│   ├── requests/       # Заявки и запросы
│   ├── monitoring/     # Автоматические уведомления от систем мониторинга
│   ├── spam/           # Спам и подозрительные письма
│   ├── info/           # Информационные рассылки
│   └── unknown/        # Не удалось классифицировать
├── tests/              # Тесты
├── email_reader.py     # Чтение и парсинг писем
├── classifier.py       # Логика классификации
├── processor.py        # Обработка: перемещение файлов, логирование
├── main.py             # Точка входа
├── run.sh              # Bash-скрипт запуска
└── README.md
```

## Категории писем

| Категория | Описание |
|-----------|----------|
| `critical` | Критические инциденты — сервис упал, всё горит |
| `requests` | Заявки: доступы, оборудование, настройки |
| `monitoring` | Автоматические алерты от систем мониторинга |
| `spam` | Спам, фишинг, подозрительные письма |
| `info` | Дайджесты, новости, информационные рассылки |
| `unknown` | Не подошло ни под одну категорию |

## Как запустить

### Требования

- Python 3.9+
- Никаких внешних библиотек — только стандартная библиотека Python

### Запуск

```bash
# Положить письма в папку inbox/
# Запустить через bash-скрипт:
bash run.sh

# Или напрямую через Python:
python main.py --inbox inbox/ --output processed/
```

### Запуск тестов

```bash
python -m pytest tests/ -v
```

## Результат работы

После запуска:
- Письма разложены по папкам в `processed/`
- В консоли — статистика по категориям
- В файле `processing.log` — подробный лог с каждым письмом
